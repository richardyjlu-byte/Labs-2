WordPress database: main tables

WordPress stores most content and configuration in a set of core tables created during a standard install. Below are the common core tables (single-site) and what each one is for.

wp_posts

Central content table. Stores posts, pages, attachments, navigation menu items, and custom post types. Key fields include an ID, author, timestamps, status, title, content, and post type.

wp_postmeta

Stores key/value "meta" for posts in wp_posts. This is how WordPress and plugins attach extra attributes (for example: featured image ID, custom fields, SEO metadata, e-commerce product details) without altering the core schema.

wp_users

User accounts table. Contains core identity fields like username (login), password hash, email, display name, registration time, and activation key.

wp_usermeta

Stores key/value metadata for users, similar to postmeta. This includes user preferences, capabilities/roles, and plugin-specific data.

wp_options

Site-wide configuration and settings. Stores WordPress core settings (site URL, blog name), theme options, and many plugin settings. Values may be "serialized" when storing structured data.

wp_comments

Stores comments left on posts (and other content types), including the comment text, author info, timestamps, approval status, and which post the comment belongs to.

wp_commentmeta

Stores key/value metadata for comments. Used by plugins or newer core features that need extra comment attributes without adding columns to wp_comments.

wp_terms

Stores the term names themselves (e.g., "Tequila", "Bourbon", "Ontario"). Terms are shared across different taxonomies.

wp_term_taxonomy

Defines what each term *means* in a specific taxonomy (category, tag, or custom taxonomy). This table links a term to a taxonomy, includes a description, parent/child relationship, and a count.

wp_term_relationships

The join table that connects posts (and other objects) to terms in a taxonomy. This is how WordPress knows which categories/tags a given post belongs to.

wp_termmeta

Stores key/value metadata for terms. Added to support attaching extra information to categories/tags/custom terms (often used by plugins).

wp_links

Legacy table from older WordPress versions for the Links/Blogroll feature. Some installations still have it for backward compatibility, but many modern sites do not actively use it.
